PHASE 1 — 建立知識庫（跑一次）

1. 文字解析與 Chunking 分塊

STEP 1

讀取文件，抽出純文字

前端類比

就像 fs.readFileSync() 讀取靜態資源，再交給 parser 處理

RAG 用途

把 PDF、Markdown、HTML 等格式，通通轉成純文字，後面才能切塊和向量化

src/index.js
import fs from 'fs'
import path from 'path'
import pdfParse from 'pdf-parse'
import dotenv from 'dotenv'
dotenv.config()

// 讀取 Markdown 文件（最簡單）
async function readMarkdown(filePath) {
  return fs.readFileSync(filePath, 'utf-8')
}

// 讀取 PDF 文件
async function readPDF(filePath) {
  const buffer = fs.readFileSync(filePath)
  const data = await pdfParse(buffer)
  return data.text  // 純文字
}

// 使用
const text = await readMarkdown('./docs/company-faq.md')
console.log(`讀取完成，共 ${text.length} 個字元`)
STEP 2

Chunking — 把長文切成小段

⚠️ 為什麼要切？

LLM 有 Token 上限（像 API request body 有大小限制）。把整篇文件丟進去太貴又不準確，切成小段能讓搜尋更精準。

src/index.js
import { RecursiveCharacterTextSplitter } from '@langchain/textsplitters'

async function chunkText(text) {
  const splitter = new RecursiveCharacterTextSplitter({
    chunkSize: 500,    // 每段最多 500 字元
    chunkOverlap: 50, // 前後重疊 50 字元（避免語意被切斷）
  })
  return splitter.splitText(text)
}

const chunks = await chunkText(text)
console.log(`切成 ${chunks.length} 段`)
console.log('第一段預覽：', chunks[0].substring(0, 100))
💡 chunkSize 怎麼選？

一般建議 300～500 字元。太小會失去上下文，太大搜尋準確度下降。你的文件是 FAQ 問答的話，可以按問題切（一個 Q&A = 一段）。

2. Embedding 向量化

STEP 3

每段文字 → 轉成向量

前端類比

就像把每個 DOM 元素轉成它在頁面上的 { top, left, width, height } 座標資訊，方便做碰撞偵測

RAG 用途

把文字轉成 1536 維的數字陣列，「語意相近」的段落在這個空間裡距離很近，後面才能用距離來搜尋

src/index.js
import OpenAI from 'openai'

const openai = new OpenAI({
  apiKey: process.env.OPENAI_API_KEY
})

async function getEmbedding(text) {
  const res = await openai.embeddings.create({
    model: 'text-embedding-3-small', // 便宜又夠用
    input: text,
  })
  return res.data[0].embedding  // [0.13, -0.27, 0.89, ...] 1536 維
}

// 批次處理所有 chunks（注意 API rate limit，加延遲）
async function embedAllChunks(chunks) {
  const results = []
  for (const [i, chunk] of chunks.entries()) {
    const vector = await getEmbedding(chunk)
    results.push({ id: i, text: chunk, vector })
    await new Promise(r => setTimeout(r, 100)) // 避免打太快
    console.log(`向量化進度：${i+1}/${chunks.length}`)
  }
  return results
}

const embeddedChunks = await embedAllChunks(chunks)
✅ 模型選擇建議

text-embedding-3-small：便宜、速度快，適合大多數場景。text-embedding-3-large：精度更高，適合對搜尋準確度要求高的場景。

STEP 5

建立 Collection，存入向量

前端類比

Collection = 資料庫的一張 Table。建立 Collection 就像 CREATE TABLE，然後 upsert 就像 INSERT

RAG 用途

把向量 + 原始文字 + metadata 一起存進去。查詢時靠向量找相關段落，顯示時用原始文字給 LLM 讀

src/index.js
import { QdrantClient } from '@qdrant/js-client-rest'

const qdrant = new QdrantClient({
  url: process.env.QDRANT_URL
})

const COLLECTION = 'my-knowledge-base'

async function setupCollection() {
  // 建立 collection（如果已存在就跳過）
  const exists = await qdrant.collectionExists(COLLECTION)
  if (!exists.exists) {
    await qdrant.createCollection(COLLECTION, {
      vectors: {
        size: 1536,          // text-embedding-3-small 的維度
        distance: 'Cosine',  // 用餘弦相似度計算距離
      }
    })
    console.log('Collection 建立完成')
  }
}

async function saveToVectorDB(embeddedChunks) {
  const points = embeddedChunks.map(chunk => ({
    id: chunk.id,
    vector: chunk.vector,
    payload: {
      text: chunk.text,       // 原始文字（給 LLM 讀）
      source: 'company-faq', // metadata：來源
    }
  }))

  await qdrant.upsert(COLLECTION, { points })
  console.log(`存入 ${points.length} 筆向量完成！`)
}

// 執行 Phase 1
await setupCollection()
await saveToVectorDB(embeddedChunks)
console.log('知識庫建立完成！')
PHASE 2 — 查詢回答（每次提問觸發）

4. 向量搜尋 + LLM 回答

STEP 6

搜尋相關段落

前端類比

就像 fetch('/api/search?q=遠距工作')，但後端是用語意相似度，不是關鍵字比對

RAG 用途

把使用者的問題也轉成向量，找出 Vector DB 裡距離最近的 Top-K 段落

src/query.js
import OpenAI from 'openai'
import { QdrantClient } from '@qdrant/js-client-rest'
import dotenv from 'dotenv'
dotenv.config()

const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY })
const qdrant = new QdrantClient({ url: process.env.QDRANT_URL })
const COLLECTION = 'my-knowledge-base'

async function searchRelevantChunks(question, topK = 3) {
  // 1. 把問題轉成向量
  const res = await openai.embeddings.create({
    model: 'text-embedding-3-small',
    input: question,
  })
  const questionVector = res.data[0].embedding

  // 2. 在 Vector DB 找最相近的段落
  const results = await qdrant.search(COLLECTION, {
    vector: questionVector,
    limit: topK,       // 取前 3 筆
    with_payload: true // 一起回傳原始文字
  })

  // 3. 取出原始文字
  return results.map(r => r.payload.text)
}
STEP 7

組合 Prompt，讓 LLM 回答

src/query.js
async function askWithRAG(question) {
  // 1. 撈相關段落
  const relevantChunks = await searchRelevantChunks(question)
  const context = relevantChunks.join('\n\n---\n\n')

  // 2. 組合 Prompt（把問題 + 相關段落打包）
  const prompt = `你是公司的 AI 助理，請根據以下資料回答問題。
如果資料中沒有相關資訊，請誠實說不知道，不要自己猜測。

【參考資料】
${context}

【問題】
${question}`

  // 3. 送給 LLM
  const completion = await openai.chat.completions.create({
    model: 'gpt-4o-mini',  // 便宜快速
    messages: [
      { role: 'user', content: prompt }
    ],
    temperature: 0.3,    // 低溫讓回答更穩定
  })

  return {
    answer: completion.choices[0].message.content,
    sources: relevantChunks,  // 回傳出處，方便 debug
  }
}

// 測試
const result = await askWithRAG('公司遠距工作的規定是什麼？')
console.log('回答：', result.answer)
console.log('參考段落：', result.sources)
✅ 這樣就完成了！

LLM 現在只能根據你塞進去的段落回答，不會再亂猜。如果問到資料庫沒有的問題，它會說「資料中沒有相關資訊」。

5. 串成 API（給前端呼叫）

可選

用 Express 包成 REST API 或 Streaming

src/server.js
import express from 'express'
import { askWithRAG } from './query.js'

const app = express()
app.use(express.json())

// 一般 REST API
app.post('/api/ask', async (req, res) => {
  const { question } = req.body
  const result = await askWithRAG(question)
  res.json(result)
})

// Streaming API（像 ChatGPT 逐字輸出）
app.post('/api/ask/stream', async (req, res) => {
  const { question } = req.body
  const chunks = await searchRelevantChunks(question)
  const context = chunks.join('\n\n')

  res.setHeader('Content-Type', 'text/event-stream')

  const stream = await openai.chat.completions.create({
    model: 'gpt-4o-mini',
    stream: true,  // 開啟 streaming
    messages: [{ role: 'user', content: `根據以下資料回答：\n${context}\n\n問題：${question}` }]
  })

  for await (const chunk of stream) {
    const token = chunk.choices[0]?.delta?.content ?? ''
    if (token) res.write(token)  // 逐字傳給前端
  }
  res.end()
})

app.listen(3001, () => console.log('RAG API 啟動於 http://localhost:3001'))
💡 前端怎麼接 Streaming

用 fetch + response.body.getReader()，每拿到一個 chunk 就更新畫面，就是 ChatGPT 那個逐字輸出的效果。

6. 常見問題 & 調校技巧

問題 原因 解法
回答不準確 Chunk 切太小，上下文不足 增加 chunkSize 到 600~800，或增加 topK 到 5
回答太慢 LLM 模型太大，或 topK 塞太多 換 gpt-4o-mini，減少 topK 或 chunkSize
AI 還是在亂猜 Prompt 沒有限制它 Prompt 加上「若資料中無相關資訊，請說不知道」
語意切斷 chunkOverlap 太小 增加 chunkOverlap 到 80~100
文件更新後沒反應 Vector DB 沒重建 文件更新後重跑 index.js（或做增量更新）
